# CharacterArc UI 设计方案

- 文档版本：`v0.1`
- 更新时间：`2026-04-29`
- 设计基准：`ui-ux-pro-max`
- 适用范围：`CharacterArc Electron 桌面端`

## 1. 设计目标

CharacterArc 的产品定位不应是“好看的小说工具”，而应是：

- 本地优先的创作工作台
- 面向长时间使用的写作 IDE
- 同时管理项目、设定、角色、大纲、正文和 AI 辅助的复合型桌面工具

因此整体 UI 的核心目标是：

- 稳定：窗口缩放、页面切换、长内容编辑时都不混乱
- 专注：突出当前任务，减少视觉噪音
- 高效：高频操作路径短，状态反馈清晰
- 可扩展：后续接入快捷键、版本 diff、关系图时不推翻结构

## 2. 总体视觉方向

### 2.1 风格定位

采用 `写作 IDE` 风格，而不是营销型产品页面风格。

参考方向：

- `VS Code` 的分栏结构与工具感
- `Scrivener` 的章节管理心智
- `Notion` 的轻编辑器可读性

关键词：

- 克制
- 安静
- 稳定
- 低装饰
- 高信息密度

### 2.2 视觉原则

- 少用大面积渐变、玻璃拟态、悬浮卡片
- 用分隔线、层级、留白和排版建立结构
- 每个页面只保留一个主操作重点
- 品牌感通过主题色、图标、排版节奏体现，而不是靠特效

### 2.3 基础审美约束

- 默认使用浅色主题
- 深色模式作为第二阶段扩展
- 圆角克制：`4px / 6px / 8px`
- 阴影最小化：只给弹层和浮出元素
- 组件状态必须明确：默认、hover、focus、active、disabled

## 3. 设计令牌

### 3.1 色彩系统

基础中性色：

- 页面背景：`#F5F5F7`
- 面板背景：`#FFFFFF`
- 弱背景：`#FAFAFA`
- 分隔线：`#D4D4D8`
- 强分隔线：`#C7C7CF`
- 正文主文字：`#202124`
- 辅助文字：`#5F6368`
- 弱提示文字：`#7A7F87`

语义色：

- Primary：继承主题色
- Success：`#15803D`
- Warning：`#A16207`
- Danger：`#DC2626`
- Info Surface：使用主色的低饱和浅背景

主题要求：

- 保留现有 `蓝 / 绿 / 橙 / 玫红`
- 主题色只影响：
  - 主按钮
  - 激活导航项
  - 焦点态
  - 选中态
  - AI 强调元素
- 不允许主题色侵入大面积背景

### 3.2 字体系统

UI 字体：

- `PingFang SC`
- `Microsoft YaHei`
- 系统无衬线备用

正文字体：

- `Noto Serif SC`
- `Source Han Serif SC`
- `Georgia`

推荐使用方式：

- 界面控件与导航使用无衬线
- 长篇正文编辑区允许使用衬线字体
- 数字统计、字数、版本号优先使用等宽数字表现

### 3.3 字号系统

统一字号阶梯：

- `12`：说明、标签、次要元信息
- `13`：辅助信息、状态
- `14`：默认控件文字
- `16`：正文基础字号
- `18`：小标题
- `20`：页面标题
- `24`：重点区域标题
- `28`：项目或章节主标题

### 3.4 间距系统

使用 `4pt` 节奏：

- `4 / 8 / 12 / 16 / 20 / 24 / 32`

规则：

- 同级元素最小间距 `8`
- 模块内部常用间距 `12` 或 `16`
- 页面区块常用间距 `20` 或 `24`

## 4. 页面级设计方案

## 4.1 项目中心

### 目标

让用户快速完成以下动作：

- 找到最近项目
- 新建项目
- 查看项目进度概况
- 进入上次创作上下文

### 页面结构

顶部区：

- 页面标题
- 项目搜索
- 排序切换
- 新建项目按钮

主体区：

- 项目列表网格
- 最近编辑优先
- 支持空状态

### 卡片信息层级

每张项目卡只保留：

- 项目名
- 题材
- 当前字数
- 最近编辑时间
- 项目状态标签

不建议保留：

- 过大的装饰封面
- 过多辅助按钮
- 二级操作入口散落在卡片表面

### 交互规则

- 单击卡片主体：进入项目
- 卡片右上角更多菜单：编辑、导出、删除
- 新建项目按钮固定为主按钮

## 4.2 新建项目向导

### 目标

降低首次创建项目的认知负担，同时保留 AI 初始化入口。

### 布局结构

- 顶部：标题 + 说明
- 中部：表单区
- 底部：取消 / 创建项目

### 表单设计规则

- 一列布局优先
- 字段顺序固定
- 长文本简介独占一行
- AI 初始化开关放在表单底部

### 状态要求

- 校验失败时错误就近显示
- 创建中按钮进入 loading
- 成功后直接进入新项目工作台

## 4.3 项目工作台

### 产品角色

项目工作台是“项目级管理中枢”，不是章节正文的主战场。

### 布局结构

- 左侧：一级导航
- 中央：当前模块内容
- 顶部：全局搜索、项目路径、当前模块名

### 左侧导航规范

导航项：

- 作品概览
- 世界观设定
- 角色图鉴
- 剧情大纲
- 章节创作
- 项目设置

规则：

- 导航项统一高度
- 激活态用左侧指示或淡色背景，不用高饱和整块填充
- 收起时只显示图标，展开时显示图标 + 名称

### 中央内容区规范

所有模块统一使用：

- `Panel Header`
- `Panel Toolbar`
- `Panel Body`

要求：

- 标题区不要过高
- 搜索、筛选、主按钮放在工具条
- 内容区域按模块类型变化

## 4.4 作品概览页

### 目标

让用户一眼知道项目现状，并快速返回关键内容。

### 模块组成

- 项目摘要
- 核心统计
- 最近编辑章节
- 重点设定 / 重点角色 / 重点大纲

### 视觉策略

- 概览页可以比其他模块稍有展示感
- 但仍保持工具化风格
- 数据卡片不超过 4 个主指标

推荐四个指标：

- 累计字数
- 章节数量
- 角色数量
- 大纲节点数量

## 4.5 世界观设定页

### 目标

支持快速浏览、增删改查和 AI 扩写。

### 结构

- 顶部：模块标题、搜索、筛选、新增按钮
- 主体：设定卡片列表
- 侧边或弹层：编辑表单

### 卡片规则

- 标题优先
- 类型标签次之
- 内容预览限制在 3 到 5 行
- 操作入口隐藏在更多菜单

### 体验重点

- 搜索要快
- 长内容不应把列表撑爆
- 编辑不跳整页，优先弹层

## 4.6 角色图鉴页

### 目标

同时支持“快速浏览角色卡”和“进入细节编辑”。

### 结构

- 顶部：搜索、筛选、新建角色、AI 生成
- 主体：角色列表网格

### 角色卡层级

- 头像或色块
- 角色名
- 身份 / 定位
- 标签
- 简介摘要

### 设计要求

- 卡片尺寸统一
- 标签数量超出时折叠
- 角色名必须强可读

## 4.7 剧情大纲页

### 目标

让“大纲按卷组织”的结构清晰易维护。

### 推荐布局

- 左：分卷列表
- 右：当前分卷的大纲节点

或：

- 上：分卷 tabs
- 下：节点列表

### 节点卡信息

- 标题
- 冲突点
- 摘要
- 目标字数

### 交互要求

- 新增分卷和新增节点的入口清晰分开
- 节点编辑尽量在当前上下文内完成
- 后续预留拖拽排序能力

## 4.8 章节创作页

### 产品级优先级

这是 CharacterArc 最核心的页面，必须做成整个产品体验最成熟的一页。

### 布局结构

- 左侧：分卷 + 章节树
- 中央：正文编辑主区
- 右侧：AI 助手 / 设定提示 / 版本信息

### 左侧章节树

信息层级：

- 分卷名
- 分卷目标字数
- 章节标题
- 章节状态
- 章节预估字数

要求：

- 当前章节高亮清晰
- 支持新增章节
- 支持拖拽排序
- 支持按关键词筛选

### 中央编辑区

结构：

- 标题输入区
- 章节元信息条
- 正文编辑器工具栏
- 正文编辑区
- 底部状态栏

编辑区要求：

- 阅读宽度控制在 `760px - 860px`
- 正文留白舒适，但不能像阅读器那样过宽
- 标题与正文之间间距明显
- 状态栏固定展示：
  - 当前字数
  - 保存状态
  - 历史版本数
  - 当前章节序号

### 右侧 AI 助手

不要做成传统聊天应用式界面，而要更像“创作辅助控制台”。

推荐结构：

- 快捷动作区
- 对话/建议区
- 插入正文入口
- 当前上下文摘要

后续可扩展为标签切换：

- AI 助手
- 设定参考
- 历史版本

### 响应式策略

- `>= 1440px`：三栏并列
- `1100px - 1439px`：AI 面板可收起或移到底部
- `820px - 1099px`：章节树缩窄，工具栏压缩
- `< 820px`：章节树优先折叠，AI 面板默认收起

## 4.9 项目设置页

### 目标

把设置做成标准桌面工具设置，而不是杂乱的信息集合。

### 分组结构

- AI 模型配置
- 存储与备份
- 导入导出
- 显示与主题

### 设计要求

- 每个分组保持卡片或分节结构
- 重要字段放上方
- 危险操作和普通操作分开

### 当前建议新增项

- 界面缩放比例
- 默认写作字体
- 默认正文字号
- 自动保存策略

## 5. 组件规范

## 5.1 按钮

按钮分为：

- Primary
- Secondary
- Ghost
- Danger

规则：

- 每个区域只允许一个 Primary
- Icon-only 按钮必须有 tooltip
- 危险操作统一红色语义

## 5.2 输入框

统一规则：

- 高度统一
- Focus 态明显
- Placeholder 只作提示，不承担标签职责

正文编辑器除外：

- 使用独立字体和行高系统
- 与表单输入框区分明显

## 5.3 标签与状态

标签只承载：

- 状态
- 类型
- 轻量分类

不承载：

- 主要导航
- 主操作

## 5.4 弹层与模态框

使用原则：

- 编辑实体：模态框或抽屉
- 删除确认：确认弹窗
- 批量操作：优先抽屉而不是新页面

## 6. 动效与反馈

### 6.1 动效原则

- 只保留必要转场
- 时长控制在 `150ms - 240ms`
- 不用装饰性大动画

### 6.2 状态反馈

必须统一的反馈包括：

- 保存中
- 保存成功
- 保存失败
- AI 生成中
- AI 返回失败
- 删除确认
- 导入导出完成

### 6.3 空状态

每个模块都要有：

- 明确说明
- 下一步动作
- 不使用过度装饰插图

## 7. 响应式与缩放策略

### 7.1 自适应原则

- 桌面优先
- 保证在小窗口下仍可工作
- 不允许出现主要编辑区不可读、不可点、被压扁

### 7.2 缩放策略

内置缩放比例：

- `75%`
- `85%`
- `100%`
- `110%`
- `125%`
- `140%`

要求：

- 缩放影响整个应用
- 与系统缩放兼容
- 设置需本地持久化

## 8. 实施优先级

### 第一阶段

- 统一设计令牌
- 统一按钮、输入框、标签、面板结构
- 清理高装饰样式

### 第二阶段

- 重做项目中心
- 重做工作台中部面板头部
- 统一设置页分组结构

### 第三阶段

- 把章节创作页打造成完整写作 IDE
- 补齐章节树、AI 助手、版本视图的层级与交互

### 第四阶段

- 深色模式
- 快捷键
- 版本 diff
- 更完整的响应式布局

## 9. 设计验收清单

上线前至少检查：

- 是否仍有不必要的大圆角、渐变、悬浮卡片
- 当前页面是否只有一个主操作重点
- 当前状态是否清晰
- 文本层级是否足够明确
- 小窗口下是否仍可完整工作
- 缩放后是否出现布局错位
- AI 助手是否仍像“工具”而不是“聊天玩具”
- 章节编辑页是否足够稳定、安静、可连续使用
