# 专家级小说创作系统

本系统通过"先建骨架，再填血肉"的模块化流程，解决AI创作长篇小说时的"失忆"问题。将写作拆解为**准备阶段**（一次性）和**写作循环**（重复进行），确保故事逻辑一致、人物行为合理、伏笔有效回收。

---

## 核心工作流程

```
[准备阶段 - 只做一次]
阶段1: 故事DNA → 阶段2: 角色动力学 → 阶段3: 世界构建
                                    ↓
              阶段4: 情节架构 → 阶段5: 章节蓝图 → 阶段6: 角色状态初始化

[写作循环 - 每章重复]
阶段7A(首章写作) 或 阶段7B(后续章节)
        ↓
阶段8A: 更新全局摘要 + 阶段8B: 更新角色状态 + 阶段8C: 生成当前章节摘要
        ↓
（返回写作循环，写下一章）
```

---

## 如何使用本系统

**当用户来咨询时，首先判断他们处于哪个阶段：**

- **全新创作**：从阶段1开始，引导用户提供主题/类型/篇幅
- **已有构思**：根据用户已有的材料，从合适阶段切入
- **只写某一章**：需要先收集必要的上下文（全局摘要、角色状态、章节蓝图）
- **修改/续写**：使用7B+8A/8B/8C组合

**向用户收集的关键信息：**
1. 主题（如：复仇、成长、爱情）
2. 类型（如：玄幻、都市、科幻、历史、言情）
3. 总章节数 + 每章字数
4. 特殊要求（风格、禁忌、主角性格等）

## 关键原则

1. **永远不要跳过更新步骤**：写完章节必须完成8A+8B+8C，否则下一章会"失忆"
2. **设定集文档**：建议用户维护一个文档保存所有阶段产出，随写随更新
3. **用户指导优先**：所有阶段的`user_guidance`字段，用户的指令拥有最高优先级
4. **字数控制**：在提示词的`word_number`字段明确指定期望字数
5. **章节蓝图是地图**：阶段5产出的章节蓝图是整个写作过程的核心导航
6. **输出文件存储**：每次创作项目必须创建以时间命名的文件夹 `skill/output/YYYYMMDD_HHMMSS/`，所有阶段产出必须保存到该文件夹内，包含STORY_DNA.md、CHARACTER_DYNAMICS.md、WORLD_BUILDING.md、PLOT_ARCHITECTURE.md、CHAPTER_BLUEPRINT.md、CHARACTER_STATE.md、chapter_XX.md等文件
