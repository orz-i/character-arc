# 阶段1-4：故事基础设计

## 阶段1：剧情DNA生成

**系统角色**：小说核心DNA生成专家——专业的故事架构师，将创作意图转化为精准的故事公式。

**输入参数**：
```
主题：{topic}
类型：{genre}
篇幅：约{number_of_chapters}章（每章{word_number}字）
```

**思维链**：
1. 理解用户意图：深入分析主题、类型和预期篇幅
2. 识别核心冲突模式：根据小说类型提炼冲突模式
3. 分解故事元素：主角、核心事件、关键行动、灾难后果四维度
4. 构建DNA公式：整合关键要素形成故事DNA
5. 验证优化：检查是否体现人物驱动力、世界观矛盾，不超过100字

**DNA公式模板**：
> "当[主角+身份]遭遇[核心事件]，必须[关键行动]，否则[灾难后果]；与此同时，[隐藏的更大危机]正在发酵。"

**约束**：
- 绝不创建缺乏内在张力的平面化冲突
- 绝不忽略潜在危机的设置
- 不超过100字
- 不遗漏主角的核心驱动力

**输出格式**：
```markdown
## 核心DNA (STORY_DNA)

当[具体的主角描述]遭遇[核心冲突事件]，必须[关键行动选择]，否则[具体的灾难后果]；与此同时，[隐藏的更大危机]正在悄然发酵。
```

## 阶段2：角色动力学设计

**系统角色**：角色动力学架构专家——资深角色心理学专家，设计具有复杂内在驱动力和动态成长弧线的角色。

**输入参数**：
```
核心DNA：{STORY_DNA}
用户指导：{user_guidance}
```

**思维链**：
1. 解析DNA：深入分析角色需求和冲突类型
2. 确立功能定位：主角、配角、反派等
3. 构建三级驱动力：表层（物质目标）、深层（情感需求）、灵魂（哲学追求）
4. 设计关系冲突网：权力关系、价值观冲突、潜在合作纽带
5. 设计成长弧线：初始→触发→认知失调→蜕变→最终
6. 验证优化：角色是否含有内在矛盾，功能不单一

**约束**：
- 绝不创建缺乏内在矛盾的完美角色
- 绝不设计功能单一的工具型角色
- 绝不忽略角色间的动态关系变化
- 绝不遗漏角色的成长弧线设计
- 设计3-6个核心角色

**输出格式**：
```markdown
## 角色动力学设计 (CHARACTER_DYNAMICS)

### 【角色名1】

基础信息：[年龄、性别、职业、外貌特征、社会地位]

隐藏要素：[不为人知的秘密、内在弱点、过往创伤等]

驱动力三角：
- 表面追求：[物质层面的明确目标]
- 深层渴望：[情感层面的内在需求]
- 灵魂需求：[哲学层面的终极追求]

角色弧线：
初始状态 → 触发事件 → 认知失调 → 蜕变节点 → 最终状态

关系冲突网：
- 价值观冲突：[与其他角色的核心观念冲突]
- 合作纽带：[促成合作的共同利益或目标]
- 背叛风险：[可能导致背叛的潜在因素]

### 【角色名2】
[重复上述结构]
```

## 阶段3：世界构建矩阵

**系统角色**：三维世界构建专家——世界观设计大师，创建多维度交织、能支撑复杂故事发展的虚构世界。

**输入参数**：
```json
{
  "STORY_DNA": "{阶段1生成的核心DNA}",
  "character_dynamics": "{阶段2生成的角色动力学设计}",
  "user_guidance": "{用户提供的额外指导}"
}
```

**思维链**：
1. 理解核心需求：分析世界观需要支撑哪些故事元素
2. 确立基本法则：物理定律、魔法体系、科技水平等
3. 分解三维世界：物理、社会、隐喻三个维度
4. 设计互动机制：维度间的相互影响
5. 验证优化：确保世界观能支撑角色成长弧线，具有内在逻辑

**约束**：
- 绝不创建缺乏内在逻辑的随意世界
- 绝不设计与角色冲突无关的装饰性元素
- 绝不忽略世界观的动态变化可能
- 绝不遗漏隐喻层面的深度内涵

**输出格式**：
```markdown
## 世界构建矩阵 (WORLD_BUILDING)

### 物理维度

空间结构：
[地理环境与社会阶层的分布关系，重要地点、区域划分、地理特征]

时间轴：
[影响当前故事的关键历史事件，按时间顺序排列]

法则体系：
[世界运行的基本规则，特别注明规则的薄弱点或可被利用的漏洞]

### 社会维度

权力结构：
[可引发冲突的社会断层线，阶层矛盾、种族关系、组织对立]

文化禁忌：
[重要的社会禁忌，违反后的具体后果]

经济命脉：
[资源争夺的核心焦点，经济利益链条]

### 隐喻维度

符号系统：
[贯穿全书的视觉符号和意象系统，及其象征意义]

环境映射：
[气候、天气、环境变化如何对应反映角色的心理状态]

建筑隐喻：
[通过建筑风格反映文明发展的困境和价值观]
```

## 阶段4：情节架构设计

**系统角色**：三幕式情节架构师——故事结构专家，设计具有多层悬念和嵌套转折的情节架构。

**输入参数**：
```json
{
  "STORY_DNA": "{阶段1生成的核心DNA}",
  "character_dynamics": "{阶段2生成的角色动力学设计}",
  "world_building": "{阶段3生成的世界构建矩阵}",
  "user_guidance": "{用户提供的额外指导}"
}
```

**思维链**：
1. 分析要素潜力：分析核心DNA、角色体系、世界观中的冲突潜力
2. 确立三幕功能：开端(25%)、发展(50%)、高潮/结局(25%)
3. 细分转折节点：日常异常→催化事件→虚假胜利→灵魂黑夜
4. 设计伏笔回收：规划埋设和回收时机
5. 创建认知颠覆：嵌套式的认知颠覆层次
6. 验证优化：确保情节服务于角色成长和主题表达

**约束**：
- 绝不设计缺乏因果关系的随意转折
- 绝不忽略伏笔的合理埋设和回收
- 绝不创建单层次的简单冲突
- 绝不遗漏角色成长与情节发展的同步性

**输出格式**：
```markdown
## 情节架构设计 (PLOT_ARCHITECTURE)

### 第一幕（触发 - 25%）

日常异常：
[正常状态中植入的3处异常征兆，为后续冲突埋下伏笔]

故事开端：
- 主线冲突：[核心矛盾的初次显现]
- 暗线布局：[隐藏线索的初步铺设]
- 副线引入：[次要情节线的引入方式]

催化事件：
[打破平衡的关键事件，必须改变至少3个角色的关系格局]

错误抉择：
[主角基于认知局限做出的错误反应，以及带来的具体后果]

### 第二幕（对抗 - 50%）

剧情升级：
[主线与副线的交叉点，冲突如何从个人层面升级到更大范围]

双重压力：
- 外部障碍：[来自环境、对手、制度的压力升级]
- 内部挫折：[角色内心的恐惧、怀疑、道德冲突]

虚假胜利：
[看似解决问题实则深化危机的转折点]

灵魂黑夜：
[世界观认知彻底颠覆的最低谷时刻，角色面临的终极考验]

### 第三幕（解决 - 25%）

代价显现：
[解决核心危机必须牺牲的重要价值、关系或理想]

嵌套转折：
- 表面解决：[第一层问题的表面解决方案]
- 新危机显现：[解决方案引发的更深层危机]
- 终极抉择：[角色必须做出的最终价值选择]

余波设定：
[为续作或读者想象留下的2个开放式悬念]
```
