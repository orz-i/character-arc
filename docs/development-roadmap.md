# CharacterArc 开发排期

- 文档版本：`v0.1`
- 更新时间：`2026-04-28`
- 适用范围：`CharacterArc 桌面端项目`

## 一、排期目标

本排期用于将当前已经具备高保真原型交互的桌面应用，逐步推进为具备真实本地存储、真实 AI 能力和稳定导入导出的可用版本。

当前项目状态可概括为：

- UI 框架已基本成型
- 主要模块已具备基础 CRUD
- 导入导出入口已存在
- 多数 AI 功能仍为 mock
- 本地存储仍以轻量状态存储为主

因此后续排期重点不是“先堆功能”，而是优先完成数据层、AI 层和工具链闭环。

## 二、阶段划分

建议拆成三个阶段推进：

1. 第一阶段：可用基础版
2. 第二阶段：创作增强版
3. 第三阶段：桌面工具完善版

---

## 第一阶段：可用基础版

### 目标

让 CharacterArc 从“可运行原型”升级为“稳定可本地使用的创作工具基础版”。

### 核心任务

#### 1. 本地数据层落地

- 引入正式本地数据存储
- 将当前 `localStorage` 迁移到更稳定的数据层
- 保证以下数据可持久化：
  - 项目信息
  - 世界观词条
  - 角色
  - 大纲节点
  - 章节
  - 设置项

#### 2. 项目数据结构稳定化

- 统一项目导入导出结构
- 明确 JSON 导出格式
- 增加字段版本兼容策略

#### 3. 导入 JSON 校验

- 校验文件是否为合法 JSON
- 校验核心字段是否存在
- 校验章节、角色、大纲结构是否完整
- 失败时给出明确错误提示

#### 4. 章节管理补齐

- 支持章节重命名
- 支持章节删除后的稳定回退逻辑
- 支持章节排序
- 保证章节状态切换不丢内容

#### 5. 项目管理补齐

- 项目编辑信息稳定化
- 项目封面上传或本地设置
- 项目删除后状态稳定

### 第一阶段验收标准

- 关闭应用再打开后，数据完整恢复
- 导入合法 JSON 可成功恢复项目
- 非法 JSON 会提示错误而不是静默失败
- 章节新增、删除、排序、编辑可稳定使用
- 项目基本管理闭环成立

---

## 第二阶段：创作增强版

### 目标

让应用从“本地内容管理工具”升级为“真正可辅助创作的 AI 创作工具”。

### 核心任务

#### 1. 真实 AI Provider 接入

- 接入 OpenAI 兼容接口
- 接入 DeepSeek / Anthropic / 其他可配置提供商
- 将设置页中的模型配置真正接入调用链

#### 2. 替换 mock AI 功能

将以下功能由 mock 改为真实 AI 请求：

- 世界观扩写
- 角色生成
- 大纲扩写
- 章节润色建议

#### 3. 章节创作增强

- AI 润色结果写回正文
- 设定查阅与章节联动
- 章节元信息扩展
  - 标题
  - 摘要
  - 状态
  - 预估字数

#### 4. 概览页增强

- 展示更真实的项目统计
- 展示最近编辑章节
- 展示重点设定/重点角色/重点节点

#### 5. 搜索增强

- 由当前“面板内筛选”升级为项目级搜索
- 聚合世界观 / 角色 / 大纲 / 章节结果

### 第二阶段验收标准

- 设置页配置的模型可以实际调用
- 世界观、角色、大纲、章节辅助不再是 mock
- 章节可通过 AI 进行真实润色或建议生成
- 搜索能跨模块工作
- 概览页具备真实项目导航价值

---

## 第三阶段：桌面工具完善版

### 目标

让产品从“可用创作工具”提升到“成熟桌面应用”。

### 核心任务

#### 1. 导出体验增强

- JSON 导出结构标准化
- TXT 导出排版增强
- 支持按模块导出
  - 仅角色
  - 仅大纲
  - 仅章节

#### 2. 桌面工具能力增强

- 日志查看能力
- 本地错误诊断
- 最近项目列表增强
- 更细的自动保存策略

#### 3. 交互与效率工具

- 快捷键
- 更细粒度的菜单操作
- 更统一的状态反馈
- 更顺滑的动画与过渡

#### 4. 后续扩展基础

- 为关系图预留结构
- 为 Prompt 模板深化预留结构
- 为写作风格系统深化预留结构

### 第三阶段验收标准

- 导出结果更适合真实使用
- 桌面应用具备基础诊断与可维护能力
- 高频创作操作具备效率工具支持
- 产品结构可继续向高级功能扩展

---

## 三、建议开发顺序

建议按以下顺序推进，避免返工：

1. 本地数据层
2. 导入导出格式与校验
3. 章节与项目管理补齐
4. 真实 AI 接入
5. 概览页和搜索增强
6. 桌面工具能力增强

---
